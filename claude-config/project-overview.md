## 2. 프로젝트 목적
어린이가 터치로 자신만의 캐릭터를 만들고, AI를 통해 동화를 생성하며,
아빠·엄마·할머니·할아버지의 목소리로 동화를 읽어주는 **가족 동화 앱**.

### 핵심 가치
- 어린이의 **창의력** 자극 (캐릭터 제작, 동화 선택)
- **가족의 목소리**로 읽어주는 따뜻한 경험 (아빠, 엄마, 할머니, 할아버지)
- AI 기반 **맞춤 동화 생성**
- 어린이 친화적 **터치 UX**

## 3. 앱 구조 — 하단 네비게이션 (5개 메뉴)

```
┌──────────────────────────────────────────────┐
│                  콘텐츠 영역                  │
├────────┬────────┬────(홈)────┬────────┬───────┤
│내 캐릭터│  동화  │  ● 홈 ●   │ 찜목록 │  마이  │
└────────┴────────┴───────────┴────────┴───────┘
```
> 홈 버튼은 가운데 돌출형 핑크 원형 버튼 (#FFA7A7)

### 메뉴 1. 내 캐릭터 (My Character)
- 어린이가 터치로 자신만의 캐릭터를 제작
- 캐릭터 파츠 선택 방식 (RPG 캐릭터 생성과 유사)
  - 머리형, 눈, 코, 입, 헤어, 피부색, 의상 등
- 완성된 캐릭터는 **동화 만들기 메뉴에서 삽화로 활용** 가능
- 여러 캐릭터 저장 및 관리

### 메뉴 2. 동화 (Making a Fairy Tale)
- 만들 동화의 **카테고리 선택** (모험, 우정, 가족, 판타지 등)
- **'동화 만들기' 버튼**으로 AI 동화 생성 요청
- 생성된 동화에 내 캐릭터 삽입 가능
- 완성된 동화는 저장하거나 홈 메뉴에 공유 가능
- **AI 동화 출력 형식**: 슬라이드 형식 또는 영상 재생 형식 중 선택

### 메뉴 3. 홈 (Home) ← 중앙 돌출 버튼
- 앱 메인 화면: **모두의 동화제작소**
- 탭 구성: 이야기 / 그림조각
- 태그 기반 필터 (예: #프렌치스타일, #꽃 ...)
- 섹션 구성
  - **주제별 모음**: 테마 카드 가로 스크롤
  - **신규 출시 목록**: 최신 동화 그리드
  - **추천 목록**: 추천 동화 카드
- 각 동화 상세에서 목소리 선택 후 읽기(TTS) 기능
  - 목소리 종류: 아빠 / 엄마 / 할머니 / 할아버지

### 메뉴 4. 찜목록 (Favorites)
- 기본 동화, AI 동화 등에서 찜한 동화 모아보기
- 오프라인에서도 접근 가능하도록 로컬 저장 지원 (예정)

### 메뉴 5. 마이 (My Page)
- 사용자 프로필 및 설정
- 내가 만든 동화 목록
- 가족 목소리 등록 관리 (예정)
- **언어 설정**: 한국어 / 日本語 선택 (앱 설정 섹션)

## 4. 핵심 기능 상세

### TTS (Text-to-Speech) — 가족 목소리
| 목소리 | 특징 |
|--------|------|
| 아빠 | 낮고 안정적인 남성 목소리 |
| 엄마 | 따뜻하고 부드러운 여성 목소리 |
| 할머니 | 정감 있는 노인 여성 목소리 |
| 할아버지 | 근엄하고 포근한 노인 남성 목소리 |
> 추후 실제 가족의 목소리를 녹음해 등록하는 기능 고려

### AI 동화 생성
- 사용자가 선택한 카테고리 + 캐릭터 정보를 기반으로 AI가 동화 텍스트 생성
- 백엔드 API를 통해 LLM(대형 언어 모델) 호출

### AI 동화 출력 형식 — 슬라이드 vs 영상

AI가 생성한 동화는 두 가지 형식으로 제공된다.

#### 형식 1. 슬라이드 형식 (Slide Format)
그림책처럼 페이지 단위로 넘기는 방식.

| 항목 | 내용 |
|------|------|
| UI 구현 | `PageView` + `PageController` 기반 전체화면 슬라이드 |
| 페이지 구성 | 상단 삽화(이미지) + 하단 텍스트 오버레이 |
| 이동 방식 | 좌우 스와이프 또는 화살표 버튼 탭 |
| TTS 연동 | 페이지 전환 시 해당 페이지 텍스트를 자동 낭독 |
| 진행 표시 | 하단 페이지 인디케이터 (dot 또는 숫자) |
| 데이터 구조 | `SlidePageModel { pageIndex, imageUrl, text }` 리스트 |
| 로컬 캐시 | Hive `slide_cache_box`에 페이지 데이터 캐싱 (오프라인 지원) |
| 서버 데이터 | 슬라이드 페이지 목록 (`GET /fairytale/{id}/slides`) |

#### 형식 2. 영상 재생 형식 (Video Playback Format)
AI가 생성한 동화를 미리 렌더링한 영상으로 제공하는 방식.

| 항목 | 내용 |
|------|------|
| UI 구현 | `video_player` 패키지 기반 전체화면 플레이어 |
| 영상 구성 | 삽화 슬라이드쇼 + 나레이션 오디오가 합성된 mp4 파일 |
| 재생 컨트롤 | 재생/일시정지, 시크바, 볼륨 조절 |
| TTS | 영상 오디오 트랙에 나레이션 내장 (별도 TTS 호출 불필요) |
| 데이터 구조 | `videoUrl: String` (서버 CDN URL) |
| 로컬 저장 | 다운로드 시 앱 로컬 파일시스템에 저장 (오프라인 재생) |
| 서버 데이터 | 영상 URL (`GET /fairytale/{id}/video`) |

#### 형식 선택 기준
```
사용자가 직접 페이지를 넘기며 읽고 싶다? → 슬라이드 형식
영상처럼 자동 재생으로 감상하고 싶다?    → 영상 재생 형식
```

#### 화면 전환 흐름
```
동화 상세 화면
  ├─ [슬라이드로 읽기] 버튼 → FairytaleSlideScreen (PageView 기반)
  └─ [영상으로 보기] 버튼   → FairytaleVideoScreen (video_player 기반)
```

### 캐릭터 제작
- 파츠(parts) 기반 레이어 합성 방식
- 각 파츠는 에셋으로 관리 (`assets/character_parts/`)
- 완성 캐릭터는 이미지로 렌더링하여 저장
